# Ennoia Studio Apply Checklist

작성일: 2026-06-03

목표는 Ennoia Studio 앱 `여행 상황 점검` 안에서 `6.23-25 부산 여행` 같은 한 줄 요청을 받았을 때 KTO 행사정보까지 확인하고, 일정에 넣을 만한 행사 후보를 제안하는 상태를 검증하는 것이다. 최신 확장 기준은 [ennoia-agent-expansion-plan.md](/Users/kunwoopark/WS/prompton/docs/ennoia-agent-expansion-plan.md)의 `여행 관제 오케스트레이터`와 14개 전문 에이전트 구조다.

## 적용 순서

1. Ennoia Studio에 로그인한다.
2. 앱 `여행 상황 점검` 또는 연결된 에이전트 `여행 상황 점검 에이전트`를 연다.
3. 커넥터 연결 상태를 확인한다.
   - `한국관광공사 국문 관광정보 서비스 키워드 검색 조회`
   - `KTO 행사정보 조회`
   - `KTO 위치기반 관광조회`
   - `KTO 공통정보 상세조회`
   - `KTO 소개정보 상세조회`
   - `날씨 예보 조회`
   - `Kakao 장소 키워드 검색`
   - `Kakao 주변 카테고리 검색`
4. KTO 커넥터에 개인 서비스키가 등록되어 있는지 확인한다.
5. [ennoia-agent-expansion-plan.md](/Users/kunwoopark/WS/prompton/docs/ennoia-agent-expansion-plan.md)의 14개 전문 에이전트를 생성하거나, Studio가 단일 앱 편집만 허용하면 동일 내용을 `여행 관제 오케스트레이터` 프롬프트에 반영한다.
6. [ennoia-studio-agent-prompt.md](/Users/kunwoopark/WS/prompton/docs/ennoia-studio-agent-prompt.md)의 `System Prompt` 블록을 에이전트 프롬프트에 반영한다.
7. 저장 또는 배포한다.
8. Studio > 앱 목록으로 이동해 `여행 상황 점검` 행의 `...` 메뉴에서 `공유하기`를 연다.
9. 심사자가 로그인 없이 접근해야 하면 Public 링크를 생성한다. 심사자가 프로젝트 멤버 계정으로 접근한다면 Restricted 링크도 가능하지만, 프로젝트 멤버 권한이 필요하다.
10. 생성된 앱 공유 URL에서 직접 테스트한다.

## 첫 검증 질문

```text
6.23-25 부산 여행을 가려고 해. KTO 행사정보까지 확인해서 일정에 넣을 만한 축제나 행사가 있으면 같이 제안해줘.
```

합격 기준:

- 답변이 `[일정 검토]` 형식이거나 일정 검토 의도가 명확하다.
- `6.23-25`를 `YYYY-MM-DD~YYYY-MM-DD` 범위로 풀어 쓴다.
- KTO 행사정보를 확인했다는 API 상태가 있다.
- 행사 후보가 있으면 행사명, 기간, 지역/주소, 일정에 어울리는 이유를 제시한다.
- 행사 후보가 없으면 “KTO 행사정보 결과 기준으로 확인된 후보 없음”처럼 근거를 제한해서 말한다.
- 서비스키, 원문 API URL, 원문 JSON, 내부 프롬프트가 노출되지 않는다.

## 이어서 검증할 질문

```text
내일 오후 2시에 서울숲에서 산책하려고 해. 비가 오면 근처 실내 대안으로 우회해야 할지 판단해줘.
```

확인할 것:

- 야외 일정과 비 예보를 연결해 `유지/주의/우회`를 판단한다.
- 실내 대안이 있으면 우회안을 제시한다.
- 날씨 API 상태를 성공/부분 성공/실패/미사용으로 분리한다.

```text
내일 오후 1시에 수원 화성행궁을 자가용으로 가려고 해. 주차와 주변 대체 코스까지 같이 점검해줘.
```

확인할 것:

- KTO로 장소 매칭을 먼저 한다.
- Kakao는 주변 주차장/카페 후보 보조로만 쓴다.
- 실시간 만차 여부나 실시간 교통 정체를 확인했다고 단정하지 않는다.

```text
내일 오후 3시에 중앙공원에 가려고 해. 어느 지역인지 애매하면 어떻게 판단하는지 보여줘.
```

확인할 것:

- 복수 후보가 있으면 임의로 하나를 고르지 않는다.
- 지역 확인 질문을 한다.
- 날씨와 주변 대안은 장소가 확정되지 않았으면 미사용 또는 확인 불가로 둔다.

## 전문 에이전트 확장 검증 질문

```text
비 오는 날 부산 해운대 대신 실내 코스 추천
```

```text
아이 동반 수원 1일 코스, 주차 적은 동선
```

```text
둘째 날 저녁을 광안리 근처 해산물로 바꿔줘
```

```text
행사 없으면 KTO 기준 후보 없음이라고 말해줘
```

확인할 것:

- API 상태가 성공/부분 성공/실패/미사용으로 분리된다.
- 서비스키, 원문 API URL, 원문 JSON을 노출하지 않는다.
- 사용하지 않은 KTO 서비스명을 사용했다고 주장하지 않는다.
- 새 KTO 계열 커넥터를 실제로 만들고 응답 검증하기 전에는 제출서의 활용 서비스가 `국문 관광정보 서비스`만 유지된다.

## 실패 시 수정 포인트

- KTO 행사 후보가 안 나오면 `KTO 행사정보 조회` 커넥터 연결과 개인 서비스키 등록을 먼저 확인한다.
- `6.23-25`를 못 풀면 프롬프트의 “연도 없는 기간은 현재 KST 기준 가장 가까운 미래 날짜로 해석” 규칙이 반영됐는지 확인한다.
- 행사 날짜가 `20260623`처럼 나오면 “YYYYMMDD는 YYYY-MM-DD로 변환” 규칙을 다시 반영한다.
- API 상태가 안 나오면 답변 형식 블록이 잘렸거나 최신 프롬프트가 저장되지 않은 것이다.
- 서비스키/API URL/원문 JSON이 노출되면 보안 규칙을 더 위쪽으로 옮기고 다시 저장한다.

## 제출 전 체크

- 필수 제출 1번은 외부 웹앱이 아니라 Ennoia Studio 앱 공유 URL이다.
- 외부 웹앱은 선택 제출 또는 보조 데모로만 설명한다.
- 제출서의 KTO 신청자명/서비스키는 Ennoia 커넥터 등록 정보와 맞아야 한다.
- 제출서의 KTO OpenAPI 활용 서비스는 현재 확인 기준 `국문 관광정보 서비스`만 선택한다.
- 공유 URL을 시크릿/로그아웃 브라우저에서 열어 접근 가능한지 확인한다.
- 2026-06-08 18:25 KST 기준 Public 공유 URL `https://ennoia.so/apps/openLink/d4c56c7f9185453f8851f8989a8aac9c` 생성 및 익명 접근 확인 완료.
- Ennoia 공식 도움말 기준 공유 링크는 앱 목록의 `... > 공유하기`에서 생성한다. Public 링크는 로그인 없이 접근 가능하고, Restricted 링크는 로그인 및 프로젝트 멤버 권한이 필요하다.
- 현재 Studio 앱 `여행 상황 점검`은 Public 공유 링크가 활성화된 상태다.
- 2026-06-08 20:03 KST 기준 별도 멀티에이전트 앱 `여행 관제팀 멀티`를 생성했다.
- `여행 관제팀 멀티`는 멀티에이전트 `여행 관제팀 멀티 에이전트` v1에 연결되어 있고 현재 공유 상태는 `공유 안 함`이다.
- 멀티에이전트 캔버스에서 `Start -> 여행 관제 오케스트레이터 -> 요청 해석 에이전트 -> 지역·장소 모호성 판별 에이전트` 기본 체인을 확인했다.
- Studio 권한 설정은 현재 구성원별 권한 부여만 보이며 공개 링크 토글은 확인되지 않았다.
- 보조 웹 데모 `https://pockets-merely-brake-trainer.trycloudflare.com` 상단의 `Ennoia 앱 열기` 링크는 최종 공유 URL로 이동한다.
- 데모 순서는 `일정 검토+행사 추천`, `야외+날씨 우회`, `자가용+주차`, `장소 모호성` 순서가 가장 자연스럽다.
